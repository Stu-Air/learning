Arch Linux Installation Guide

Pre-installation

This document is a guide for installing Arch Linux using the live system booted from an installation medium made from an official installation image. Visit the Download page and, depending on how you want to boot, acquire the ISO file or a netboot image, and the respective GnuPG signature. On a system with GnuPG installed, do this by downloading the ISO PGP signature (under Checksums in the page Download) to the ISO directory, and verifying it with:

gpg --keyserver-options auto-key-retrieve --verify archlinux-version-x86_64.iso.sig

When the installation medium's boot loader menu appears,

  • If you used the ISO, select Arch Linux install medium and press Enter to enter the installation environment.
  • if you used the Netboot image, choose a geographically close mirror from Mirror menu, then select Boot Arch Linux and press Enter.

    • You will be logged in on the first virtual console as the root user, and presented with a Zsh shell prompt. Personally for connecting to wifi if your device doesn't have an ethernet port code below to power your device and connect to your router.

    iwctl device phy0 set-property Powered on
    iwctl device wlan0 set-property Powered on
    iwctl station [WIFI DEVICE] connect [SSID]

    Script Installation

    Once above has been completed you can start the installation process, we need to download the git repo if you have not already downloaded it. this can be done by using git.

    https://github.com/Stu-Air/Arch-Installer-NG.git

    Before runnning the script we need to fill in a few things at the top of the installer.sh file, following includes:

  • Drive you want to install Arch linux on
  • Hostname of the installed machine
  • Root password (leave blank to be prompted)
  • Main user to create (by default, added to wheel group, and others)
  • The main user's password (leave blank to be prompted)
  • System timezone
  • Keyboard layout used in system
  • Wireless device, leave blank to not use wireless and use DHCP instead
  • amd or intel chipset ucode

    • Once you have done we can proceed with the following command from inside the repo folder you just downloaded.

    sh ./installer.sh

    Extras - Desktop

    you may have noticed that there was an "extra" section. this gives you the choice of 3 options:

  • kde desktop with sddm login manager
  • xfce desktop with lightdm login manager
  • gnome desktop with gdm login manager

    • these can be install on any Arch linux based system, by the following command

    pacman -Sy --noconfirm - < /[name_of_file].txt

    Extras - Applications

    this sections is where standard repo applications are installed and als ofrom the AUR whis needs an aur helper this section installs the paru helper. by default the script will install my personal applications from github repo (https://github.com/Stu-Air/arch-apps).
    To install paru on any Arch linux based distro with the following command

    cd ~
    git clone https://aur.archlinux.org/paru-bin.git
    chown $USER_NAME:$USER_NAME ~
    chmod -R 777 paru-bin
    cd paru-bin
    echo -en "$USER_PASSWORD" |sudo -u $USER_NAME makepkg -si --noconfirm

    when installing applications from the aur with paru the commands change very slighty. we will be using the following command:

    paru S- [packageName]

    when instll from the Arch Linux repo's the command uses pacman.

    pacman S- [packageName]

    here is some of my default packages that will be installed if you decided you fill in this section at the top

  • discord
  • mpv
  • thunderbird
  • libreoffice-fresh
  • firefox
  • steam

    • to run this command on any Arch Linux based distro :

    sh ./applications.sh
